    • Al utilizar instrucciones o ciclos anidados.

def fact_recur(n):

Búsqueda binaria (bisección)¶

  • Con una lista ordenada:
  1. Escogemos un ndice i que divide la lista a la mitad.
  2. Revisamos si L[i] == e:
  3. Si no, revisamos si L[i] es mayor o menor que e
  4. Dependiendo de la respuesta, buscamos a la izquierda o derecha de L.
  • Aproximacin de divide y conquista.
    • Obtenemos una versin ms pequea del problema.

Búsqueda binaria (bisección)¶

  • En cada paso, la lista se divide a la mitad.
  • Terminamos de ver la lista cuando $n/2^i = 1$, es decir, $i = \log_{2} n$.
  • Complejidad del problema es $O(\log n)$, donde $n = \text{len}(L)$.

def bisect_search1(L, e):
    # Si L es vaca
    if L == []:
        return False
    # Si L es de largo 1
    elif len(L) == 1:
        return L[0] == e
    else:
        # Encontramos el "centro" de la lista
        half = len(L)//2
        
        # Si el valor del centro es mayor a e
        if L[half] > e:
            # Bsqueda en la parte superior
            return bisect_search1( L[:half], e)
        else:
            # Bsqueda en la parte inferior
            return bisect_search1( L[half:], e)

Implementación 1¶

  • Complejidad $O(\log n)$ para cada llamada a biseccin.

def bisect_search2(L, e):
    def bisect_search_helper(L, e, low, high):
        if high == low:
            return L[low] == e
        
        mid = (low + high)//2
        if L[mid] == e:
            return True
        elif L[mid] > e:
            if low == mid: #nothing left to search
                return False
            else:
                return bisect_search_helper(L, e, low, mid - 1)
        else:
            return bisect_search_helper(L, e, mid + 1, high)
    
    # Cuerpo de la funcin
    if len(L) == 0:
        return False
    else:
        return bisect_search_helper(L, e, 0, len(L) - 1)

Complejidad de las torres de Hanoi¶

  • El nmero de pasos para resolver una torre de $n$ discos es $T_n$, donde $$T_n = 2T_{n-1} + 1$$

    • Basta con ver los pasos con 2 y 3 discos para ver por qu.

  • Al iterar hacia atrs: $$ \begin{eqnarray} T_n = & 2T_{n-1} + 1\\ = & 2(2T_{n-2} + 1) + 1 \\ = & 4T_{n-2} + 2 + 1 \\ = & 4(T_{n-3} + 1) + 2 + 1 \\ = & 8T_{n-3} + 4 + 2 + 1 \\ = & \vdots \\ = & 2^k T_{n-k} + 2^{k-1} + \ldots + 4 + 2 + 1 \\ = & 2^{n-1}\underbrace{T_1}{=1} + 2^{n-2} + \ldots + 4 + 2 + 1 \\ = & 2^{n} - 1 \\ \end{eqnarray} $$

  • Por lo tanto, el orden de crecimiento de este algoritmo es $O(2^n)$.

Complejidad exponencial¶

  • Dado un conjunto de enteros, queremos generar el conjunto potencia:

  • El conjunto $\{1,2,3,4\}$ generara $$\{\},\{1\},\{2\},\{3\},\{4\},\{1,2\},\{1,3\},\{1,4\},\{2,3\},\{2,4\},\{3,4\}, \{1,2,3\},\{1,2,4\},\{1,3,4\},\{2,3,4\},\{1,2,3,4\}$$

  • Descripcin recursiva: conjunto potencia de $n$ elementos es la unin del potencia del conjunto con $n-1$ elementos al incluir elemento $n$ y al excluir al elemento $n$.

  • Complejidad $O(2^n)$.

def genSubsets(L):
    # Conjunto potencia del conjunto vaco
    if len(L) == 0:
        return [[]] 
    
    # Generar todos los conjuntos sin el ltimo elemento
    smaller = genSubsets(L[:-1]) 
    
    # Crear una lista con el ltimo elemento
    extra = L[-1:] 
    new = []
    
    # Para cada conjunto del conjunto ms pequeo
    for small in smaller:
        # Agregar a las soluciones pequeas  
        # el ltimo elemento
        new.append(small+extra) 
    
    # Combinar conjuntos con y sin ltimo elemento
    return smaller + new
