def linear_search(L, e):
    found = False
    for i in range(len(L)):
        if e == L[i]:
            found = True
    return found

  • Se debe buscar a travs de todos los elementos para decidir que el elemento est ausente.

  • $O(\text{len}(L))$ para el ciclo $\times O(1)$ para determinar if e == L[i]

  • Complejidad general: $O(n)$, donde $n = \text{L}$.

Búsqueda binaria (recapitulación)¶

  • Con una lista ordenada:
  1. Escogemos un ndice i que divide la lista a la mitad.
  2. Revisamos si L[i] == e:
  3. Si no, revisamos si L[i] es mayor o menor que e
  4. Dependiendo de la respuesta, buscamos a la izquierda o derecha de L.
  • Aproximacin de divide y conquista.
    • Obtenemos una versin ms pequea del problema.

Búsqueda binaria (bisección)¶

  • En cada paso, la lista se divide a la mitad.
  • Terminamos de ver la lista cuando $n/2^i = 1$, es decir, $i = \log_{2} n$.
  • Complejidad del problema es $O(\log n)$, donde $n = \text{len}(L)$.

  • La complejidad de la recursin seguir siendo $O(\log n)$, donde $n = \text{L}$.

def bisect_search2(L, e):
    def bisect_search_helper(L, e, low, high):
        if high == low:
            return L[low] == e
        
        mid = (low + high)//2
        if L[mid] == e:
            return True
        elif L[mid] > e:
            if low == mid: #nothing left to search
                return False
            else:
                return bisect_search_helper(L, e, low, mid - 1)
        else:
            return bisect_search_helper(L, e, mid + 1, high)
    
    # Cuerpo de la funcin
    if len(L) == 0:
        return False

Implementación 2¶

  • Al utilizar la funcin helper evitamos la copia.

    • Pasamos ndices low y high en cada llamada.

  • Como la lista no se copia, se pasa por "referencia" hacia las llamadas sucesivas tenemos $O(1)$ en las nuevas llamadas.

  • Ahora $O(\log n)*O(1) = O(\log n)$

Búsqueda en una lista ordenada¶

  • Con bsqueda lineal, la complejidad es $O(n)$.

  • Con bsqueda binaria, se puede buscar un elemento con $O(\log n)$.

    • Es necesario que la lista est ordenada.

  • Por qu hace sentido ordenar primero y despus buscar?

    • Ordenar nunca tiene complejidad menor a $O(n)$, es ms eficiente ordenar primero para amortizar el costo sobre muchas bsquedas.

    • Si $K$ es el nmero de bsquedas: $$\text{ORDENAR} + K*O(\log n) < K * O(n)$$

      • Para $K$ muy grande, el tiempo de $\text{ORDENAR}$ se vuelve irrelevante, si el costo de ordenar es suficientemente pequeo.

Algoritmos de ordenamiento¶

  • Queremos ordenar eficientemente una lista de entradas (tpicamente nmeros).

  • Veamos algunos mtodos.

Monkey sort¶

  • Bogosort (alias permutation sort, stupid sort) es un algoritmo de ordenamiento aleatorio.

  • Ordenar una pila de cartas:

    • Arrojarlas al aire y recogerlas

    • Estn ordenadas?

    • Si no, repetir.

  • Mejor caso: $O(n)$ para revisar si la lista est ordenada.

  • Peor de los casos: $O(.)$ indeterminado (muy mala suerte!)

Método de burbuja¶

  • Compara pares consecutivos de elementos.

  • Intercambia los elementos para que el menor vaya primero.

  • Cuando se llega al final de la lista, empezamos otra vez.

  • Paramos hasta que no existan intercambios.

def bubble_sort(L):
    swap = False
    while not swap:
        swap = True
        for j in range(1, len(L)):
            if L[j-1] > L[j]:
                swap = False
                temp = L[j]
                L[j] = L[j-1]
                L[j-1] = temp

  • Ciclo interior para las comparaciones.

  • Ciclo exterior para los los mltiples pasos

  • Complejidad $O(n^2)$ donde $n = \text{len}(L)$

    • $\text{len}(L)-1$ comparaciones y $\text{len}(L)-1 $ barridas.

Merge sort¶

  • Utilizamos "divide y conquista".

  • Si la lista es de largo 0 o 1, ya est ordenada!

  • Si tiene ms elementos, la dividimos en dos listas y ordenamos cada una.

  • Combinamos las sublistas ordenadas.

    • Miramos el primer elemento de cada una, ponemos el menor al final de los resultados ordenados.

    • Cuando una lista est vaca, copiamos el resto de la otra lista.

def merge(left, right):
    result = []
    i,j = 0,0
    while i < len(left) and j < len(right):
        if left[i] < right[j]:
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            j += 1
    while (i < len(left)):
        result.append(left[i])
        i += 1
    while (j < len(right)):
        result.append(right[j])
        j += 1
    return result

Merge sort¶

  • Se divide la lista sucesivamente en mitades.

  • Se conquistan piezas ms pequeas antes de las piezas ms grandes.

def merge_sort(L):
    # Caso base
    if len(L) < 2:
        return L[:]
    else:
        middle = len(L)//2
        # Divide
        left = merge_sort(L[:middle])
        right = merge_sort(L[middle:])
        # Vencers con el paso de merge
        return merge(left, right)

Complejidad de merge sort¶

  • En el primer nivel de recursin:

    • $n/2$ elementos en cada lista

    • $O(n) + O(n) = O(n)$, donde $n=\text{len}(L)$

  • En el segundo nivel de recursin:

    • $n/4$ elementos en cada lista

    • Dos uniones $\rightarrow O(n)$, donde $n=\text{len}(L)$.

  • Cada nivel de recursin es $O(n)$

  • Al dividir la lista por la mitad con cada llamada tenemos $O(\log n)$ por cada llamada recursiva.

  • Complejidad $O(n\log n)$, donde $n=\text{len}(L)$.

Quicksort¶

  • Divide y vencers

  • Se escoge un pivote para dividir la lista en dos sublistas.

    • La lista izquierda tiene los elementos menores que el pivote.

    • La lista derecha tiene los elementos mayores que el pivote.

  • Recursivamente, se ordenan las dos sublistas

  • Se combinan los resultados.

def quicksort(arr):
    if len(arr) <= 1:
        return arr
    pivot = arr[len(arr) // 2]
    left = [x for x in arr if x < pivot]
    middle = [x for x in arr if x == pivot]
    right = [x for x in arr if x > pivot]
    return quicksort(left) + middle + quicksort(right)

Complejidad del quicksort¶

  • Depende de la posicin del pivote elegido.

    • En el mejor caso termina al centro de la lista, dividindola en dos sublistas de igual tamao. En este caso, quicksort es $O(n \log n)$.

    • En el peor caso, el pivote termina en un extremo. En este caso, quicksort es $O(n^2)$.

    • En promedio, entonces $O(n\log n)$.

  • Existen optimizaciones que se enfocan en la eleccin del pivote.

Algoritmos de ordenamiento¶

  • Bogosort: $O(\cdot)$ no acotado

  • Burbuja: $O(n^2)$

  • Merge sort: $O(n\log n)$

  • Quicksort: $O(n \log n)$ en promedio
